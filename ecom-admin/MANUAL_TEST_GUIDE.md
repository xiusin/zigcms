# 质量中心前端手动测试指南

## 测试准备

### 1. 启动开发服务器
```bash
cd ecom-admin
npm run dev
```

### 2. 打开浏览器
访问: http://localhost:5173

### 3. 登录系统
使用测试账号登录

### 二、表单提交测试 (20 分钟)

#### 2.1 创建测试用例
1. 点击"新建用例"按钮
2. 填写表单:
   - 标题: "测试用例 001"
   - 项目: 选择任意项目
   - 模块: 选择任意模块
   - 优先级: 高
   - 前置条件: "用户已登录"
   - 测试步骤: "1. 打开页面\n2. 点击按钮"
   - 预期结果: "操作成功"
3. 点击"提交"
4. 验证:
   - [ ] 表单验证正常
   - [ ] 提交成功提示显示
   - [ ] 列表自动刷新
   - [ ] 新用例出现在列表中

#### 2.2 编辑测试用例
1. 点击某个测试用例的"编辑"按钮
2. 修改标题为"测试用例 001 (已修改)"
3. 点击"提交"
4. 验证:
   - [ ] 数据回填正常
   - [ ] 提交成功
   - [ ] 列表更新

#### 2.3 创建项目
1. 访问项目管理页面
2. 点击"新建项目"
3. 填写表单并提交
4. 验证:
   - [ ] 表单验证正常
   - [ ] 提交成功
   - [ ] 新项目卡片显示

#### 2.4 创建模块
1. 访问模块管理页面
2. 点击"新建模块"
3. 填写表单并提交
4. 验证:
   - [ ] 层级深度限制验证
   - [ ] 提交成功
   - [ ] 树形结构更新

6. 验证:
   - [ ] 弹窗关闭

5. 验证:
   - [ ] 表单提交

6. 按 `Esc`
7. 验证:
   - [ ] 取消操作，关闭表单

#### 6.3 列表快捷键
1. 在测试用例列表中

   - [ ] 选择行切换

6. 验证:
   - [ ] 显示确认对话框

   - [ ] 确认按钮为红色

#### 8.3 悬停状态
1. 悬停在卡片上

   - [ ] 卡片阴影加深
   - [ ] 无布局偏移

3. 悬停在按钮上
4. 验证:
   - [ ] 按钮颜色变化
   - [ ] 无布局偏移

5. 悬停在表格行上
6. 验证:
   - [ ] 行背景色变化
   - [ ] 无布局偏移

#### 8.4 焦点状态
1. 使用 Tab 键导航
2. 验证:
   - [ ] 表单输入框焦点样式清晰
   - [ ] 按钮焦点样式清晰
