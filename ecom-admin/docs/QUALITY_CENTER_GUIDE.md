# 质量中心使用指南

## 📊 功能概览

质量中心是融合**自动化测试系统**与**反馈系统**的统一质量管理平台，提供全方位的质量数据洞察和智能化测试管理。

## 🎯 核心功能

### 1. 质量总览 Dashboard (`/quality-center/dashboard`)

#### 1.1 统计卡片（8项核心指标）
- **测试通过率**：实时展示测试通过率，颜色动态变化（≥90%绿色，≥70%橙色，<70%红色）
- **总测试任务**：当前系统中的测试任务总数
- **活跃Bug**：未修复的Bug数量
- **待处理反馈**：等待处理的用户反馈数
- **AI修复率**：AI自动修复Bug的成功率
- **本周执行**：本周测试执行次数统计
- **反馈转任务**：反馈转换为测试任务的数量
- **平均修复时长**：Bug从发现到修复的平均时长（小时）

#### 1.2 AI质量洞察
- **风险预警**：高风险模块自动识别
- **优化建议**：基于AI分析的测试覆盖建议
- **异常检测**：性能异常、接口异常自动发现
- **趋势分析**：质量趋势向好/向坏预测
- **一键跳转**：点击洞察卡片直接跳转到相关页面

#### 1.3 质量趋势图（ECharts折线图）
- **多维度数据**：通过率、Bug数、反馈数、执行次数
- **时间周期切换**：支持近7天/30天/90天
- **交互钻取**：点击日期查看该日详细数据
- **数据缩放**：超过30天自动启用数据缩放滑块
- **双Y轴展示**：通过率（左轴）、数量（右轴）

#### 1.4 模块质量分布
- **模块维度**：按业务模块展示质量指标
- **通过率进度条**：可视化展示每个模块的测试通过率
- **Bug数标签**：颜色标识（>5红色，>2橙色，≤2绿色）
- **用例/反馈统计**：完整的质量数据关联

#### 1.5 Bug类型分布（ECharts饼图）
- **6大类型**：功能错误、界面问题、性能问题、安全问题、数据问题、逻辑错误
- **环形饼图**：美观的数据可视化
- **点击交互**：点击类型跳转到Bug列表并自动筛选
- **图例联动**：点击图例显示/隐藏对应数据

#### 1.6 反馈状态分布（ECharts环形图）
- **5种状态**：待处理、处理中、已解决、已关闭、已拒绝
- **渐变色设计**：不同状态使用不同颜色标识
- **点击钻取**：点击状态跳转到反馈列表并筛选
- **实时更新**：状态变化自动刷新

#### 1.7 活动流时间线
- **8种活动类型**：测试通过/失败、Bug发现/修复、反馈创建/解决、AI分析/修复
- **类型筛选**：下拉选择特定类型活动
- **用户信息**：显示操作人头像和姓名
- **时间排序**：最新活动优先展示

#### 1.8 关联记录
- **4种关联类型**：反馈→任务、Bug→反馈、任务→Bug、用例→Bug
- **双向追踪**：完整的数据流转路径
- **快速跳转**：点击记录跳转到详情页
- **批量操作**：支持反馈转任务、Bug转反馈

### 2. 关联追踪 (`/quality-center/link-records`)

#### 2.1 筛选功能
- **来源类型筛选**：反馈/Bug/任务/用例
- **关联类型筛选**：4种关联方向
- **重置功能**：一键清空所有筛选条件

#### 2.2 操作功能
- **反馈转任务**：选择反馈创建测试任务，支持AI自动生成用例
- **Bug转反馈**：将Bug同步到反馈系统
- **批量导出**：导出关联记录报表

#### 2.3 数据展示
- **关联方向可视化**：来源→目标箭头展示
- **创建人信息**：头像+姓名
- **时间排序**：最新关联优先

### 3. AI生成测试用例增强

#### 3.1 实时进度展示（AIGenerateProgress组件）
- **5步骤流程**：
  1. 分析生成目标
  2. 构建AI提示词
  3. 调用AI模型生成
  4. 解析生成结果
  5. 验证用例有效性
- **进度条动画**：平滑的进度更新
- **实时统计**：已生成数量、耗时、Token消耗
- **错误处理**：生成失败自动提示

#### 3.2 生成结果预览
- **用例列表**：表格展示所有生成的用例
- **字段完整**：名称、类型、方法、端点、预期状态、描述
- **批量选择**：支持全选/反选
- **单个操作**：编辑、测试运行、删除

#### 3.3 批量保存
- **选中保存**：只保存勾选的用例
- **全部保存**：一键保存所有生成的用例
- **保存前验证**：检查必填字段完整性

## 🔧 技术实现

### 1. 技术栈
- **Vue 3** + Composition API + `<script setup>`
- **Arco Design Vue**：UI组件库
- **ECharts 5**：图表可视化
- **vue-echarts**：ECharts Vue封装
- **Pinia**：状态管理
- **TypeScript**：类型安全

### 2. 核心组件
```
src/views/quality-center/
├── dashboard/index.vue              # Dashboard主页
├── link-records/index.vue           # 关联追踪
└── components/
    ├── QualityTrendChart.vue        # 趋势折线图
    ├── BugDistributionChart.vue     # Bug饼图
    ├── FeedbackDistributionChart.vue # 反馈环形图
    ├── FeedbackToTaskModal.vue      # 反馈转任务弹窗
    ├── BugToFeedbackModal.vue       # Bug转反馈弹窗
    ├── AIGenerateProgress.vue       # AI生成进度
    └── DashboardSkeleton.vue        # 骨架屏
```

### 3. 数据流
```
用户操作 → Dashboard组件
         ↓
    Pinia Store (useQualityCenterStore)
         ↓
    API调用 (src/api/quality-center.ts)
         ↓
    Mock数据 (src/mock/quality-center.ts)
         ↓
    响应数据返回 → 更新Store → 更新UI
```

### 4. 日志追踪
所有关键操作都有日志记录：
```typescript
console.log('[质量中心][操作类型][参数]', data);
```

## 🚀 使用流程

### 场景1：查看质量概览
1. 访问 `/quality-center/dashboard`
2. 查看8项核心指标
3. 查看AI洞察，点击跳转到相关页面
4. 切换趋势周期（7/30/90天）
5. 点击图表进行数据钻取

### 场景2：反馈转测试任务
1. Dashboard点击"反馈转任务"按钮
2. 选择反馈（支持搜索）
3. 填写任务名称、类型、优先级
4. 开启AI自动生成用例（可选）
5. 设置生成数量（1-50）
6. 提交创建
7. 查看AI生成进度
8. 预览生成结果
9. 批量保存用例

### 场景3：Bug同步到反馈
1. Dashboard或关联追踪页点击"Bug转反馈"
2. 选择Bug
3. 填写反馈标题、类型、优先级
4. 提交创建
5. 系统自动建立关联关系

### 场景4：追踪关联关系
1. 访问 `/quality-center/link-records`
2. 筛选来源类型/关联类型
3. 查看完整的数据流转路径
4. 点击记录跳转到详情页

## 📝 最佳实践

1. **定期查看Dashboard**：每日查看质量趋势和AI洞察
2. **及时处理高风险洞察**：优先处理红色风险预警
3. **利用AI生成用例**：新功能开发后使用AI快速生成测试用例
4. **建立反馈-测试闭环**：用户反馈及时转为测试任务
5. **追踪关联关系**：通过关联记录了解问题根源
