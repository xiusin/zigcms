# 内存泄漏与生命周期基础知识（宏观篇）

## 基础知识
- 泄漏定义：分配未释放或仍不可达，但占用资源。
- 生命周期：从分配到释放，必须覆盖使用期，过短导致悬空，过长导致泄漏。
- 所有权：谁分配谁释放；跨层/跨线程要交接责任。

## 核心概念
- **内存泄漏**：程序分配了内存但未在可达路径中释放，导致可用内存持续减少。
- **生命周期**：对象从分配到释放的时间范围，必须匹配作用域或业务需求。
- **所有权**：谁分配谁释放；跨层传递时需明确释放责任。

## 常见场景与陷阱
- **悬空指针/切片**：返回指向已结束作用域的栈内存或临时缓冲，访问即未定义行为。
- **重复分配未释放**：循环内分配、缓存/池未回收，长时间运行内存飙升。
- **异常路径缺失释放**：早退、错误分支未覆盖释放逻辑。
- **跨协程/线程传递**：未约定所有权或多次释放，导致泄漏或双重释放。
- **全局缓存/单例**：缺乏上限或淘汰策略，长期运行即泄漏风险。
- **外部资源**：文件句柄、网络连接、锁未释放同样构成“资源泄漏”。

## 识别与定位
- **监控指标**：RSS、堆内存使用、对象数、GC/分配次数（按语言工具）。
- **采样/剖析**：使用内存分析器/火焰图/heap dump，对比快照定位增长点。
- **分阶段对比**：启动基线 vs 长时运行；压测前后 diff 找热区。
- **日志与审计**：关键分配/释放点打点，标注请求 ID、协程/线程 ID。

## 预防策略
- **作用域管理**：优先栈内存，作用域结束自动回收；堆内存需明确释放时机。
- **RAII/守卫模式**：defer/errdefer/析构器确保异常路径也释放。
- **所有权约定**：API 层文档写明返回值是否需释放；传递跨层时交接责任。
- **池与缓存**：设置上限、超时、淘汰策略，定期巡检统计。
- **不可变数据**：复用只读常量，减少重复分配。
- **分层治理**：接口层（说明释放责任）、领域层（少分配）、基础设施层（池化+监控）。

## 修复流程（Checklist）
1) 复现泄漏：压测/长跑，确认指标持续上涨。
2) 采集证据：heap dump/内存剖析/日志，确定热点代码路径。
3) 定位责任：分配点 vs 未释放点，检查异常分支。
4) 制定修复：添加释放、改用栈内存、引入池化或限流、缩短生命周期。
5) 回归验证：重复压测，对比泄漏指标下降；补充自动化测试或监控告警。
- 6) 复盘沉淀：总结规则、补文档/linters，防止同类问题再现。

## 编码约定示例（语言无关）
- 函数文档需声明：返回值是否需要调用方释放；跨线程是否安全。
- 对外导出 API：必须标注所有权和生命周期；返回堆数据要提供释放函数或约定 allocator。
- 异常安全：每条分支均覆盖释放（守卫/try-finally/errdefer）。
- 长循环/守护进程：周期性自检或重置缓存，避免无界增长。

## 监控与告警
- 建立内存水位告警（绝对值 + 增长率）。
- 关键池/缓存统计：命中率、占用、淘汰次数、创建/销毁次数。
- 定期压测回归，防止“暗漏”随业务增长放大。

## 参考操作
- 如果必须返回动态数据：提供 `*_owned` 或 `free*` 接口，调用方明确释放。
- 如果只需短期使用：返回包含内部缓冲的结构体/对象，限制其作用域使用。
- 不要在公共 API 返回指向局部的裸指针/切片。

## 日常防漏清单（易记版）
- 作用域优先：能放栈上就不放堆上。
- 谁分配谁释放：跨层传递要写清释放责任。
- 守卫释放：`defer/errdefer` 或等价机制覆盖异常路径。
- 循环慎分配：大循环里禁止无界分配，能复用就复用。
- 池/缓存要上限：设置容量+淘汰策略，定期统计。
- 返回值注明：API 文档明确“是否需要释放”。
- 不要返回局部指针/切片：避免悬空引用。

## 故事记忆法（搬家/寄存/钥匙/盘点）
- **搬家箱子=结构体返回**：把物品装箱一起送到对方，箱子还在手上，里面的东西就安全（`buf` 跟着结构体存活）。搬完记得丢箱子（作用域结束自动回收）。
- **寄存柜=堆上副本**：要长期放东西就租寄存柜（堆内存），到期要续费/退租（释放），否则一直占位收费。
- **钥匙失效=悬空指针**：朋友搬走后，你手里的旧钥匙开不了门（指向已回收的栈内存），再用就出事，不要留这种钥匙。
- **盘点仓库=监控告警**：定期清点、超量告警，发现爆仓及时清理，防止堆积成灾。

## 示例对比

### 1) 悬空切片（错误示例）
```zig
fn bad() []const u8 {
    var buf: [16]u8 = undefined;
    const s = "hello";
    std.mem.copy(u8, &buf, s);
    return buf[0..s.len]; // 返回指向栈内存，调用处即悬空
}
```

### 2) 资源未释放（错误示例）
```zig
fn badFile() !void {
    var file = try std.fs.cwd().createFile("tmp", .{});
    // defer file.close(); // 缺失，异常/正常均泄漏句柄
    try file.writeAll("data");
}
```

### 3) 守卫释放（正确示例）
```zig
fn goodFile() !void {
    var file = try std.fs.cwd().createFile("tmp", .{});
    defer file.close();
    try file.writeAll("data");
}
```
